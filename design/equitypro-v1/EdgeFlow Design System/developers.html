Authentication

Every request must include a Bearer token in the Authorization header. Tokens are scoped per workspace and rotated automatically every 90 days.

# Curl example
curl https://api.equitypro.in/v3/quote/RELIANCE \
  -H "Authorization: Bearer eqp_live_a1b2c3..." \
  -H "Accept: application/json"

WebSocket streaming

Subscribe to tick data, depth updates, or aggregate index quotes. The connection uses bidirectional JSON frames over wss://stream.equitypro.in/v3.

// JS client
const ws = new WebSocket("wss://stream.equitypro.in/v3?token=eqp_live_...");
ws.onopen = () => ws.send(JSON.stringify({
  action: "subscribe",
  channels: ["ticks:RELIANCE", "ticks:TCS", "index:NIFTY50"]
}));
ws.onmessage = (e) => console.log(JSON.parse(e.data));

Webhook signatures

Verify each webhook by computing HMAC-SHA256(body, secret) and comparing with the X-EQP-Signature header. Reject anything older than 5 minutes (replay protection).
